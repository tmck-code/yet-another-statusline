Startup was the cost — and it's down.

~95% of runtime was import/startup, not rendering. Deferring the conditional imports (subprocess, shutil, tomllib) plus killing an import-time Config.load() cut a clean-machine render to 39.8 ms (from 47 ms on 3.15, ~70 ms on the original 3.13). import yas.app now pulls none of the three.

What changed

  • render/text.py — subprocess deferred into terminal_width() behind a $TMUX_PANE guard; shutil.get_terminal_size → os.get_terminal_size (kills the bz2/lzma/zlib pull).
  • info/git.py — subprocess deferred into _dirty().
  • config.py — tomllib deferred into _load_toml(), imported only after a yas.toml read succeeds.
  • tokens.py — the hidden one: TokenRate.WINDOW = _token_window() ran a full Config.load() (and tomllib) at import time. Made lazy + @functools.cached — this is what lets the tomllib deferral actually land.
  • test_config.py · test_git_info.py — updated to not couple to removed module-level names (config.tomllib, git.subprocess); use importlib.util.find_spec / sys.modules patching instead.
Python 3.15 is itself a big win: ~70 ms (3.13) → ~47–55 ms before any edits, partly because 3.15's dataclasses lazy-loads inspect — so the original ~5.5 ms dataclasses→inspect chain is largely gone, making the "eliminate dataclasses" idea (IMP-4) low-value now.

Heavy stdlib pulls

The render path is imported eagerly via app.py → config.py, info/*, layout.py. Conditional imports (used only sometimes) were the deferral targets — all three are now removed from the import path.

How this was measured

Harness: ops/profile_statusline.py (two cProfile scopes) + -X importtime + a stash-based wall-clock A/B (best-of-50 median, identical env each side). Raw artifacts in ops/profile-out/.

# cold path: imports + one render (per-module import cost)
cat ops/session-info-example.json | python3 ops/profile_statusline.py full

# steady-state: render() ×200, imports warmed outside the profiled region
cat ops/session-info-example.json | python3 ops/profile_statusline.py render

# per-module import timing
cat ops/session-info-example.json | PYTHONPATH=claude \
  python3 -X importtime claude/statusline_command.py >/dev/null 2> importtime.txt

# confirm the deferral: import path no longer pulls the three
PYTHONPATH=claude python3 -X importtime -c "import yas.app" \
  | grep -E ' (subprocess|shutil|tomllib)$'   # → (no output)
Caveat: wall-clock at the shell's 10 ms granularity can't see a 6 ms win — hence the in-process best-of-50 median timer and the deterministic -X importtime cumulative as the primary signals.

What's left, in order

  1. REN-1 — collapse the token-rate triple-read to one parse. Halves render, cheap.
  2. REN-3 — memoise Config.load() for the process (render + main both call it).
  3. REN-4 — hoist _dim_for_col out of the per-column loop (matters on wide terminals).
  4. IMP-4 — now low-value on 3.15; revisit only if you must support 3.13.
